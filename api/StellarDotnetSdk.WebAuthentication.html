ReadChallengeTransaction(Transaction, string, string, string, Network?, DateTimeOffset?)

Read a SEP 10 challenge transaction and return the client account id. Performs the following checks:

  1. Transaction sequence number is 0
  2. Transaction source account is serverAccountId
  3. Transaction has one operation only, of type ManageDataOperation
  4. The ManageDataOperation name and value are correct
  5. Transaction time bounds are still valid
  6. Transaction is signed by server

VerifyChallengeTransaction(Transaction, string, string, string, Network?, DateTimeOffset?)

Verify that a transaction is a valid Stellar Web Authentication transaction. Performs the following checks:

  1. Transaction sequence number is 0
  2. Transaction source account is serverAccountId
  3. Transaction has one operation only, of type ManageDataOperation
  4. The ManageDataOperation name and value are correct
  5. Transaction time bounds are still valid
  6. Transaction is signed by server and client

VerifyChallengeTransactionSigners(Transaction, string, ICollection<string>, string, string, Network?, DateTimeOffset?)

Verify that all signers of a SEP 10 transaction are accounted for. A transaction is verified if it signed by the server account, and all other signatures match the provided signers. Additional signers can be provided that do not have a signature, but all signatures must be matched to a signer for verification to succeed. If verification succeeds, the list of signers that were found is returned, excluding the server account id.
